## 1. ระบบติดตามสถานะงาน (state machine)

งานแต่ละชิ้นไหลตามสถานะนี้ (เหมือนกระดาน Kanban):

```
มอบงาน (Backlog) → ออกแบบ (In Progress) → รอรีวิว (In Review)
      → [ ขอแก้ ↺ กลับไป In Progress ]  หรือ  [ อนุมัติ (Approved) ]
      → รายงาน Discord → เสร็จ (Done, เก็บไฟล์)
```

- **Pixel** (UX/UI) ออกแบบ → **Cora** (Team Lead) รีวิว/อนุมัติ → **Milo** (Coordinator) รายงาน
- ไม่มีงาน = ทีม "หลับ zzz"
- งานเข้าได้ทาง **แชทกับ Claude** ("มอบงาน: ...") → เข้า inbox → แดชบอร์ดดึงไปทำเอง

## 2. สถาปัตยกรรม

| ส่วน | ไฟล์ | หน้าที่ |
|---|---|---|
| แดชบอร์ด | `index.html` | UI ทั้งหมด (พิกเซลอาร์ต 2D + แผงข้อมูล) — เปิดด้วยเบราว์เซอร์ |
| สมอง AI (ฟรี) | `server/ollama-server.js` | คุยกับโมเดลโลคัล (Ollama) |
| สมอง AI (จริง) | `server/server.js` | คุยกับ Claude API |
| คลังงานเสร็จ | `server/deliverables/*.md` | ไฟล์ผลงานจริงที่เซฟลงดิสก์ |

## 3. Roadmap (สเปกฟีเจอร์ — ชี้ section แล้วสั่งทำได้)

> ทำแล้วทำไหน ดูสถานะจริงใน `PROGRESS.md`

- **3.1 แดชบอร์ดพิกเซลอาร์ต** — แผนที่ออฟฟิศ 2D, ตัวละครเดินได้, ธีมนีออน
- **3.2 ระบบสถานะงาน** — flow ตาม section 1, การ์ดงาน, แถบความคืบหน้า
- **3.3 ต่อ AI จริง** — backend (Ollama ฟรี / Claude) + โหมดจำลอง fallback
- **3.4 มอบงานผ่านแชท** — inbox bridge (พิมพ์บอก Claude → งานเข้าระบบ)
- **3.5 คลังงานเสร็จ** — แท็บ Done + เซฟไฟล์ deliverable
- **3.6 โปรไฟล์ + สกิลตัวละคร** — modal เมื่อคลิกตัวละคร
- **3.7 ภาษาไทย** — UI + ผลลัพธ์ AI เป็นไทย

## 3.8 เก็บงานลง SQLite ✅
backend เก็บ **inbox (งานที่มอบ) + deliverables (งานเสร็จ)** ลงไฟล์ `server/data/studio-pixel.db`
ด้วย `node:sqlite` (built-in, ออฟไลน์) — ข้อมูลรอด restart · `server/db.js` · ล้างด้วย `npm run reset-db`
